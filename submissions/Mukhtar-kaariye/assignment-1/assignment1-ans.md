Research Assignment: Introduction to Data Science and Machine Learning
1. Define Data Science and Machine Learning. What is the relationship between them?
Data Science

Data Science is the process of collecting, organizing, analyzing, and interpreting data to generate useful insights and support decision-making (IBM, n.d.; AWS, n.d.).

Machine Learning

Machine Learning is a branch of artificial intelligence that enables computers to learn patterns from data and make predictions without being explicitly programmed for every task (IBM, n.d.; Google Cloud, n.d.).

Relationship Between Data Science and Machine Learning

Machine Learning is a tool used within Data Science. Data Science covers the entire process of working with data, while Machine Learning focuses on building models that learn from data and make predictions (IBM, n.d.).

Example

An online shopping company can analyze customer purchase data and use Machine Learning to recommend products that customers may want to buy. Data Science manages the overall process, while Machine Learning performs the prediction task (Google Cloud, n.d.).

2. Describe the Data Science Lifecycle

The Data Science lifecycle consists of several stages that transform raw data into useful solutions (AWS, n.d.).

Stage	                Description
Problem Definition	Identify the problem and objectives.
Data Collection	    Gather relevant data.
Data Preparation	Clean and organize the data.
Data Analysis	    Explore patterns and trends.
Model Building	    Create analytical or ML models.
Model Evaluation	Measure model performance.
Deployment	        Implement the solution.
Monitoring	        Maintain and improve the system.

Where Does Machine Learning Fit?

Machine Learning mainly fits in the Model Building and Model Evaluation stages because algorithms are trained and tested using prepared data before deployment (Google Cloud, n.d.).

3. Compare Supervised Learning and Unsupervised Learning
Feature	Supervised Learning	Unsupervised Learning
Data	Labeled	Unlabeled
Goal	Predict outcomes	Find patterns
Example	Loan approval prediction	Customer segmentation
Supervised Learning

Supervised learning uses labeled data to train models that predict outcomes. For example, a bank may predict whether a customer will repay a loan (IBM, n.d.).

Unsupervised Learning

Unsupervised learning uses unlabeled data to discover hidden patterns. For example, a company may group customers according to purchasing behavior (IBM, n.d.).

4. What Causes Overfitting? How Can It Be Prevented?

Overfitting occurs when a model learns the training data too closely, including noise and unnecessary details, resulting in poor performance on new data (AWS, n.d.).

Common causes include:

Small datasets
Complex models
Too many features
Excessive training

Prevention methods include:

Using more data
Feature selection
Cross-validation
Regularization
Early stopping (AWS, n.d.)
5. Explain How Training Data and Test Data Are Split

Training data is used to teach a machine learning model, while test data is used to evaluate its performance on unseen data (Google Cloud, n.d.).

A common split is:

Training Data	Test Data
80%	               20%

This process is important because it helps measure the model's true performance and reduces the risk of overfitting (IBM, n.d.).

6. Case Study: Netflix Recommendation System (Business)
Background

Netflix uses Data Science and Machine Learning to recommend movies and television shows based on user preferences and viewing history (Netflix Technology Blog, n.d.).

Application

Netflix collects user data such as watch history, ratings, and searches. Machine learning algorithms analyze this information and recommend content that users are likely to enjoy (Netflix Technology Blog, n.d.).

Findings

The recommendation system improves user engagement, helps customers discover content, and increases customer retention. Personalized recommendations influence a large portion of the content watched on Netflix (Netflix Technology Blog, n.d.).
